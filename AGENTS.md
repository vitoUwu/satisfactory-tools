## Overview

Bun-workspace monorepo scaffolded with Better-T-Stack: a fullstack TanStack Start (React 19 + TanStack Router, SSR via Nitro/Vite) app backed by SQLite/Turso through Drizzle ORM. There is no separate backend — the web app is fullstack ("self" backend). No test runner or linter is configured.

## Commands

Package manager is **bun** (`bun install`). Run from the repo root:

- `bun run dev` — start all apps in dev mode (web on http://localhost:3001)
- `bun run dev:web` — start only the web app
- `bun run build` — build all apps
- `bun run check-types` — TypeScript type check across workspaces (`tsc --noEmit`)

Database (proxies into `packages/db`, which loads env from `apps/web/.env`):

- `bun run db:local` — start a local SQLite DB via `turso dev --db-file local.db`
- `bun run db:push` — push schema to the database (drizzle-kit)
- `bun run db:generate` / `bun run db:migrate` — generate / run migrations
- `bun run db:studio` — open Drizzle Studio

Docker: `bun run docker:build` / `docker:up` / `docker:logs` / `docker:down` (config in `docker-compose.yml`, Dockerfiles in `apps/*/Dockerfile`).

## Architecture

```
apps/web/                 # Fullstack TanStack Start app
  src/routes/             # File-based routes (__root.tsx, index.tsx, ...)
  src/routeTree.gen.ts    # GENERATED by TanStack Router — never edit by hand
  src/router.tsx          # Router factory (getRouter)
  vite.config.ts          # tailwindcss + tanstackStart + nitro + react plugins; port 3001
packages/
  db/                     # @satisfactory-tools/db — Drizzle ORM + libsql client
    src/schema/           # Drizzle schema (drizzle-kit reads this dir)
    src/migrations/       # Generated migrations
  env/                    # @satisfactory-tools/env — t3-oss/env-core validated env vars
    src/server.ts         # DATABASE_URL, CORS_ORIGIN, NODE_ENV
    src/web.ts            # client vars (VITE_ prefix)
  ui/                     # @satisfactory-tools/ui — shared shadcn/ui components
  config/                 # @satisfactory-tools/config — shared tsconfig.base.json
```

Key points:

- Workspace packages export TypeScript source directly (no build step); depend on them with `workspace:*` and shared versions via the root `package.json` `catalog:`.
- Environment variables live in `apps/web/.env` and must be declared in the zod schemas in `packages/env` before use — access them via `@satisfactory-tools/env/server` or `/web`, never raw `process.env`.
- Database access: import `db` from `@satisfactory-tools/db`; schema tables go in `packages/db/src/schema/index.ts` (currently empty). Dialect is Turso/libsql.
- UI components are shadcn/ui primitives (built on @base-ui/react) shared through `packages/ui`. Import as `@satisfactory-tools/ui/components/<name>`. Design tokens/global styles: `packages/ui/src/styles/globals.css`. Add new shared primitives from the repo root with `npx shadcn@latest add <component> -c packages/ui`; app-specific blocks are added from `apps/web` instead. A shadcn skill is vendored in `.agents/skills/shadcn`.
- TypeScript is strict (`noUncheckedIndexedAccess`, `noUnusedLocals`, `verbatimModuleSyntax`) via `packages/config/tsconfig.base.json`.
